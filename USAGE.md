# Docshot — Інструкція користувача

Docshot — це Chrome-розширення для створення анотованих скріншотів та відео з AI-генерованими описами, які автоматично зберігаються у відповідні поля WordPress-документації.

## 1. Встановлення

> **Передумова:** Розширення не опубліковано у Chrome Web Store — встановлюється вручну.

1. Отримай папку з розширенням у колеги або з репозиторію (`build/chrome-mv3-prod`).
2. Відкрий Chrome → адресний рядок → `chrome://extensions`.
3. Увімкни **"Режим розробника"** (перемикач у правому верхньому куті).
4. Натисни **"Завантажити розпаковане розширення"** → вибери папку `build/chrome-mv3-prod`.
5. Іконка Docshot з'явиться на панелі інструментів браузера.

## 2. Авторизація

1. Натисни іконку Docshot на панелі браузера — відкриється попап.
   Попап використовує фірмову палітру: чорний фон `#020202`, темний блок `#0F0F0F`, червоний акцент `#E70000`, білий текст `#FFFFFF` і сірий текст/деталі `#c2c2c2`.
   У верхній частині попапа відображається логотип Redstone у фірмових кольорах (червоний `#E70000`, сірий `#c2c2c2`) та напис `QA` сірим кольором `#c2c2c2` в одному візуальному розмірі з логотипом.
2. Введи **WordPress логін** (email або ім'я користувача).
3. Введи **пароль застосунку** (формат: `XXXX XXXX XXXX XXXX XXXX XXXX`).

> **Як отримати пароль застосунку:**
> WordPress Адмін → Профіль → розділ "Паролі застосунків" → введи назву (наприклад "Docshot") → натисни "Додати" → скопіюй згенерований пароль.

4. Натисни **"Увійти"**.

## 3. Налаштування Gemini API ключа

Для AI-генерації описів потрібен ключ Google Gemini API. Встановлюється один раз.

1. Відкрий попап розширення.
2. Натисни **"⚙ Налаштування Gemini"** — розділ розгорнеться.
3. Встав свій ключ у поле **"API ключ"** (починається з `AIza…`).
4. Натисни **"Зберегти"** — у заголовку розділу з'явиться зелена галочка.
5. За бажанням вибери **модель** зі списку (за замовчуванням — 3.1 Flash Lite).

> **Де отримати ключ:** [Google AI Studio](https://aistudio.google.com/app/apikey) → "Create API key".

### Доступні моделі

| Модель         | Ідентифікатор          |
| -------------- | ---------------------- |
| 3.1 Flash Lite | gemini-3.1-flash-lite  |
| 3 Flash        | gemini-3-flash-preview |
| 2.5 Flash Lite | gemini-2.5-flash-lite  |
| 2.5 Flash      | gemini-2.5-flash       |
| 2.0 Flash      | gemini-2.0-flash       |

Модель можна змінити у будь-який момент — нова модель буде використовуватися з наступного запиту до AI.

## 4. Структура документації (Tab Editor)

Tab Editor — це головний інтерфейс для перегляду та редагування структури документа у WordPress.

### Відкрити Tab Editor

Натисни на **бейдж дестинації** у попапі (синій або помаранчевий блок під email).

### Вибрати документ

1. У пошуку введи назву документа — список оновлюється автоматично.
2. Натисни на потрібний документ у списку.

### Структура документа

Документ складається з:

- **Розділів** (H3) — основні блоки документа
- **Підрозділів** (H4) — вкладені елементи всередині розділу

### Дії з розділами та підрозділами

| Дія              | Як виконати                            |
| ---------------- | -------------------------------------- |
| Перейменувати    | Натисни іконку олівця поруч з назвою   |
| Видалити         | Натисни іконку ✕                       |
| Змінити порядок  | Перетягни рядок за іконку ⠿            |
| Додати підрозділ | Натисни **"+ Підрозділ"** під розділом |
| Додати розділ    | Натисни **"+ Розділ"** внизу списку    |

### Зберегти зміни структури

Після редагування структури з'явиться кнопка **"Зберегти"** — натисни її щоб зберегти зміни у WordPress.

## 5. Керування документами

Усі дії з документами доступні з екрана вибору документа (Post Selector).

### Перейменувати документ

1. Знайди документ у списку.
2. Натисни іконку олівця справа від назви.
3. Введи нову назву — підтвердь клавішею **Enter** або клікни поза полем. Для скасування — **Escape**.

> Назва зберігається у WordPress одразу після підтвердження.

### Дублювати документ

1. Знайди документ у списку.
2. Натисни іконку копії справа від назви.
3. Новий документ з'явиться у списку з префіксом **"Copy of …"**.

> Дублювання копіює документ як **чернетку** (статус `draft` у WordPress). Щоб документ став доступним для читачів, його потрібно опублікувати — див. нижче.

### Опублікувати документ

Новостворений або продубльований документ має статус **чернетки** і не видимий на сайті. Щоб опублікувати:

1. Відкрий документ у Tab Editor.
2. У вкрхній панелі з'явиться кнопка **"Опублікувати"** (відображається лише для чернеток).
3. Натисни її — документ одразу стане публічним.

### Зберегти зміни структури

Зміни розділів та підрозділів (додавання, видалення, перейменування, порядок) зберігаються локально до явного підтвердження.

Коли є незбережені зміни — у Tab Editor з'являється кнопка **"Зберегти"** у нижній панелі.

1. Внеси всі потрібні зміни структури.
2. Натисни **"Зберегти"** — зміни відправляться у WordPress.

> Поки зміни не збережено, кнопка вибору дестинації (Обрати) заблокована.

## 6. Вибір місця збереження (Destination)

Перш ніж знімати скріншот чи відео, треба вказати, **в який розділ документа** буде збережено контент.

1. Відкрий Tab Editor (натисни бейдж дестинації у попапі).
2. Знайди та відкрий потрібний документ.
3. Натисни на розділ або підрозділ, куди хочеш зберегти контент — він підсвітиться.
4. Кнопка **"Обрати"** стане активною — натисни її.

Після вибору:

- Попап покаже **синій бейдж** з назвою документа та шляхом до обраного розділу.
- Кнопки **"Скріншот"** та **"Відео"** стануть активними.

## 7. Захоплення скріншоту

### Крок 1 — Зробити скріншот

1. Перейди на вкладку браузера, яку хочеш зафіксувати.
2. Відкрий попап → натисни **"📸 Скріншот"**.
3. На сторінці з'явиться оверлей — виділи потрібну область мишкою.

### Крок 2 — Анотувати скріншот

Відкриється екран анотатора зі скріншотом.

**Доступні інструменти:**

| Інструмент  | Призначення                   |
| ----------- | ----------------------------- |
| Стрілка     | Вказівна стрілка на елемент   |
| Текст       | Підпис на зображенні          |
| Лінія       | Пряма лінія                   |
| Прямокутник | Виділення прямокутної області |

**Додаткові налаштування:**

- **Колір:** вибери один з 5 кольорів (червоний, жовтий, зелений, синій, чорний)
- **Товщина:** регулюй повзунком

**Скасувати останню анотацію:** натисни **"↩ Скасувати"** на тулбарі.

Коли анотація готова — натисни **"Далі →"**.

### Крок 3 — Описати скріншот

Відкриється екран з текстовим полем для опису.

- **Надиктувати:** натисни іконку мікрофона у правому нижньому куті поля — браузер запитає дозвіл (дозволь). Говори, текст з'являється автоматично. Натисни іконку ще раз щоб зупинити.
- **Ввести вручну:** просто друкуй у полі.
- Обидва способи можна комбінувати — текст редагується вільно.

Коли опис готовий — натисни **"Надіслати →"**.

> **Порада:** Описуй що саме показано та які дії виконуються — це покращує якість AI-опису.

### Крок 4 — AI обробка

Розширення автоматично надсилає скріншот та транскрипцію в Gemini AI. Зачекай кілька секунд.

Якщо AI повернув помилку — з'явиться повідомлення про помилку та селектор моделей **"Спробувати з іншою моделлю"**. Вибери іншу модель зі списку та натисни **"Спробувати знову"**. Зміна моделі зберігається автоматично.

## 8. Запис відео

Відкрий попап → натисни **"🎬 Відео"**. На екрані є два режими:

### Режим "Запис" (за замовчуванням)

1. Натисни **"● Почати запис"**.
2. Вибери вікно або вкладку для запису → натисни **"Поділитися"**.
3. Виконай дії які хочеш задокументувати.
4. Натисни **"■ Зупинити"** — з'явиться попередній перегляд.
5. За бажанням введи або надиктуй **додатковий опис** — натисни іконку мікрофона у полі опису.
6. Натисни **"Надіслати →"**.

### Режим "Завантажити файл"

1. Натисни вкладку **"↑ Завантажити файл"**.
2. Натисни **"Обрати відео файл"** → вибери файл з диску (mp4, webm тощо).
3. З'явиться попередній перегляд обраного відео.
4. За бажанням введи або надиктуй **додатковий опис** — натисни іконку мікрофона у полі опису.
5. Натисни **"Надіслати →"**.

Далі — AI обробка та перегляд контенту (як у кроках 4–9 для скріншоту).

## 9. Перегляд та збереження AI-контенту

Після AI обробки відкривається екран перегляду.

### Що можна зробити:

**Редагувати текст** — клікни у поле редактора та вноси зміни напряму.

**Перекласти (опціонально):**

1. Натисни кнопку перекладу та вибери мову.
2. З'явиться два стовпці: оригінал (зліва) та переклад (справа).
3. Обидва можна редагувати перед збереженням.

**Зберегти:**

- Натисни **"Додати"** — новий контент буде дописаний у кінець існуючого вмісту розділу (скріншот завантажиться у WordPress Media Library).
- Натисни **"Замінити"** — існуючий вміст розділу буде повністю замінений новим контентом.
- Після збереження повернешся до Tab Editor, де обраний розділ буде автоматично відкритий.

**Скасувати:**

- Натисни **"Скасувати"** — зміни відхиляться, повернешся до Tab Editor.

## 10. FAQ

**Кнопки "Скріншот" та "Відео" неактивні — чому?**
Потрібно виконати дві умови: (1) вибрати дестинацію у Tab Editor, (2) зберегти Gemini API ключ.

**Де взяти пароль застосунку для WordPress?**
WordPress Адмін → вгорі праворуч натисни на своє ім'я → **"Профіль"** → розділ **"Паролі застосунків"**.

**AI генерує неточний опис або повертає помилку — що робити?**
Відредагуй текст вручну на екрані перегляду перед збереженням. Якщо виникла помилка — спробуй іншу модель прямо на екрані помилки. Якість результату залежить від чіткості голосового коментаря — намагайся детально описувати що показано на скріншоті.

**Як змінити дестинацію (вибраний розділ)?**
Натисни на синій бейдж у попапі → в Tab Editor натисни на інший розділ → **"Обрати"**.

**Розширення не реагує після перезапуску браузера?**
Відкрий `chrome://extensions` та переконайся що Docshot увімкнено. Якщо ні — увімкни і спробуй знову.

**Скріншот виходить порожнім або чорним?**
Переконайся що ти виділяєш область на активній вкладці браузера, а не на системному вікні чи іншому розширенні.
